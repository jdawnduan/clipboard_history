What Happens When You Press the Hotkey

Trace the journey from a keyboard shortcut to your clipboard history

You Already Know How to Use This

Imagine you're writing an email. You copy a URL, paste it. Copy some text, paste it. Then you realize you needed something you copied five minutes ago...

That's where clipboard history comes in.

You press Cmd+Option+V and a little popup appears with your recent copies. You click "3" and — boom — the text from 20 copies ago is back in your clipboard, ready to paste.

But what's actually happening behind the scenes? Let's find out.

But wait — Clipboard History has to be "watching" for that hotkey even when it's not visible. How does it do that?

The App That's Always Watching

When you install and start Clipboard History, it runs as a daemon — a background program that starts when your computer starts and just... waits.

It's like having a receptionist who never takes breaks. Even when you're in the middle of writing, the daemon is sitting there, quietly listening for your hotkey.

💡
Key Insight

Background daemons are how your computer does things "automatically" — they watch for events and respond when needed, without you having to open an app.

The Code That Listens

Let's look at the real code that registers the hotkey. This is from the main program:

CODE 
let hotkey = HotKey::new(
  Some(Modifiers::META | Modifiers::ALT),
  Code::KeyV
);
manager.register(hotkey)?;
PLAIN ENGLISH

Create a new hotkey object that watches for Command + Option + V...

META means Command key, ALT means Option key...

KeyV is the V key itself...

Tell the system: "Let me know when this combo is pressed"

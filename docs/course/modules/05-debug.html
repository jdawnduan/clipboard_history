When Things Go Wrong

Debugging intuition for clipboard history problems

The Three Most Common Problems

When Clipboard History doesn't work, it's usually one of these three things:

🔐
Accessibility Permissions

macOS blocks apps from simulating keyboard input without permission

🔑
Hotkey Not Registered

The app can't see your Cmd+Option+V press

💾
History Not Saving

Entries disappear after restart

Problem 1: Accessibility Permissions

When you select from the popup, the app needs to simulate pressing Cmd+V to paste. macOS blocks this by default for security reasons (otherwise, malicious apps could type things for you!).

Symptoms: The popup appears, you select an item, but nothing gets pasted.

Fix: Go to System Settings → Privacy & Security → Accessibility → enable Clipboard History (or your terminal app if running from there).

⚠️
Why This Matters

Accessibility permissions let apps control your keyboard. This is why Clipboard History needs explicit approval — it's a security feature, not a bug!

Problem 2: Hotkey Not Working

The hotkey registration can fail if another app is already using that combination.

Symptoms: Nothing happens when you press Cmd+Option+V.

Debug step: Check the console output. The app prints messages when it registers (or fails to register) the hotkey.

CONSOLE 
Starting clipboard history daemon...
Registered hotkey: Cmd+Option+V (id: 1)
Press Cmd+Option+V to show clipboard history popup
PLAIN ENGLISH

The daemon is starting...

Success! The hotkey was registered with ID 1...

Instructions for using the app

If you see an error instead of "Registered hotkey", another app might be claiming that shortcut.

Problem 3: History Not Persisting

If your clipboard history disappears after restart, the app might not have permission to write to its data folder.

Location: On macOS, history is saved to:

~/Library/Application Support/clipboard-history/history.json

Debug step: Check if the file exists and is writable:

1
Open Finder

Press Cmd+Shift+G and paste the path above

2
Check for history.json

If it exists, the app is saving correctly

3
Check file size

Empty file = nothing was saved; has content = good!

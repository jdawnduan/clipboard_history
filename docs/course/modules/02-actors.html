Meet the Components

The four main actors that make clipboard history possible

Four Components, One Mission

Every piece of software has components that work together. Clipboard History has four main ones:

📋
Clipboard Module

Talks to your computer's clipboard — reads what you copied, writes what you want to paste

📚
History Module

Stores and retrieves your clipboard history — like a librarian for your copied text

👁️
Monitor

Watches the clipboard for changes — every 500ms it checks "anything new?"

🔧
Platform Module

Handles macOS-specific stuff — permissions, hotkeys, making the app hide properly

The Clipboard Module — Reading and Writing

The Clipboard module is the simplest component — it just talks to your computer's clipboard. Here's the actual code:

CODE 
pub fn get_clipboard() -> Result<String> {
  let mut clipboard = Clipboard::new()?;
  Ok(clipboard.get_text()?)
}

pub fn set_clipboard(content: &str) {
  let mut clipboard = Clipboard::new()?;
  clipboard.set_text(content)?;
}
PLAIN ENGLISH

This function reads whatever is currently in your clipboard...

Create a new clipboard connection object...

Ask for the text content, and return it (or an error if something went wrong)

Empty line — just separating the two functions

This function writes something TO the clipboard...

Create another clipboard connection...

Put the text into the clipboard — now it's ready to paste!

The History Module — The Librarian

The History module is like a librarian. It keeps an ordered list of clipboard entries, saves them to disk, and retrieves them when asked.

Key things it does:

1
Stores entries

Each entry has the text and when it was copied

2
Saves to disk

Uses JSON format, stored in your local app data folder

3
Limits size

Keeps only the 20 most recent entries (configurable)

4
Removes duplicates

If you copy the same thing twice, only keeps the newer one

The Project Structure

Here's how the code is organized into files:

src/ The source code folder
main.rs The main program — entry point, hotkeys, UI
clipboard.rs Reads and writes to the system clipboard
history.rs Stores and retrieves clipboard history
platform/ macOS and Linux specific code
mod.rs Platform selection logic
macos.rs macOS permissions, app hiding, single-instance
linux.rs Linux display server detection
